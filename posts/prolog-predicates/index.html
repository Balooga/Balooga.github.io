Prolog Learnings: Predicates and Association of Arguments

December 22, 2020 5-minute read
Functions (or procedures) in Prolog are called "predicates", but unlike functions in other languages that can only evaluate inputs to create an output, predicates do not have defined inputs and outputs. In idiomatic Prolog, a predicate is a function that works both ways.

Overview

So what does this mean and why should we care? Being a declarative logic programming language, problems are solved differently in Prolog. It is one of the languages that every developer should have some exposure to. Modern languages have closures, classes, generics, macros, etc. Prolog supports pattern matching, backtracking, and inference and sits on top of a relational database.

Similar to functions, a predicate sum/3 can be defined to sum two arguments and return the result. In the examples below, total and Total are assigned the value of 3 as a result of evaluting sum. In Prolog, variables must start with an uppercase letter. And if you are familiar with Elixir, you will recognize the functor/arity nomenclature Prolog uses to name predicates.

The sum of two arguments in Python
1

total = sum(1, 2)
The sum of two arguments in Prolog
1

?- sum(1, 2, Total)

Notice that Prolog requires that Total is passed as an argument to sum. Because in Prolog, predicates can only return true or false. This may seem archaic at first (Prolog was invented in 1972 after all. And in France.) But this is key to how Prolog works, as a predicate defines a relationship between its arguments. Prolog then attempts to derive values for the arguments in an attempt to evaluate the predicate to true.

Prolog evaluates sum(1, 2, Total) to true by assigning 3 to Total. Simple enough. But things become interesting when 3 is passed as a parameter to the third argument of sum/3. Now Prolog returns true for sum(1,2,3), because 3 is the sum of 1 and 2. Something like 3 = sum(1, 2) which is a total syntax error in Python is actually valid syntax in Prolog due to Prologs use of "Unification" (Unification) and not regular "Assignment". Unification is really powerful and core to Prolog, as can be seen in later sections.

There is a notable difference between "is" and "=". "is" performs term evaluation and the result must be a number. "=" does not perform term evaluation. And for "is", all arguments must already have been instantiated to numbers.

Conversely, Prolog returns false for sum(1,2,5) because the sum of 1 and 2 does not equal 5 and there is no way for Prolog to make that true.

It gets better.

sum(1, Y, 3) and Prolog returns Y = 2, because for sum to evaulate to true, Y must be 2. sum(X, 2, 3) results in X = 1 for the same reason.

It gets better still.

sum(X, Y, 3) and Prolog will return all combinations of X and Y that sum to 3.

After returning an answer for X and Y, the user can enter a semicolon (;) to instruct Prolog to use backtracking to attempt to find alternate values for X and y that evaluate to true.

And even better.

For sum(X, Y, Total) and helpful Prolog will return an "infinite" number of combinations of X and Y that sum to Total.

reverse/2

To continue with another example, reverse/2 which is defined as follows; reverse/2 accepts a list of 'n' elements and returns a new list containing these elements reversed. In the examples below, reverse/2 will generate expected results when both arguments receive values, no arguments receive values, or only one argument receives a value.

In the first example, Prolog is able to evaluate the predicate reverse/2 to true as the list provided in the second argument contains all the elements of the first argument, but reversed.

Prolog
1
2

?- reverse([1,2,3], [3,2,1]).
true.

In the second example, the predicate evaluates to false because [1,2,3] is not the reverse of [3,2].

Prolog
1
2

?- reverse([1,2,3], [3,2]).
false.

In the next example, to evaluate the predicate reverse/2 to true, Prolog will create and return a new list for Output containing the elements from the first list, but reversed.

Prolog
1
2

?- reverse([1,2,3], Output).
Output = [3,2,1].

Because of the relationship between the arguments, either argument can be passed a list.

Prolog
1
2

?- reverse(Input, [3,2,1]).
Input = [1,2,3].

And when values for either argument are not provided, Prolog creates a new list of anonymous variables and returns a second list with the elements reversed (anonymous variables in Prolog start with the underscore character _).

With each successive semicolon, Prolog will provide additional answers for which the predicate reverse/2 evaulates to true. So first a list of zero elements is reversed, then a list of one element, then two, then three, etc.
